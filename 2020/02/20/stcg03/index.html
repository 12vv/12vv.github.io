2D线性变换

用一个2*2矩阵进行线性变化(Linear Transformations)。

其实用于变换的这个矩阵的列就是新空间的基。比如，上面的式子中，可以看作是新空间的基向量变为$[a_{11} \ a_{21}]$和$[a_{12} \ a_{22}]$，可以分别看作是新的x轴和y轴。下面是一些先行变换的例子。

Scaling

将$[s_{x} \ 0]$和$[0 \ s_{y}]$ 作为x和y，所以x方向被缩放了$s_x$倍，y方向被缩放了$s_y$倍。

还需注意一点，如果在偏离原点处进行缩放变换，比如在原点右上方的一个图形被扩大两倍，除此之外，它还会更加偏离原点，为了避免这种情况，每次进行缩放之前，最好先平移到原点。

Shearing

仍然可以使用之前的方法来理解，比如使用$shear-x(s)$将$[1 \ 0]$和$[s \ 1]$ 作为x和y，那么x方向不发生改变，而y方向原本垂直于x的轴变为一条倾斜的轴，如果s=1，那么这条新的轴和x呈45度角。$shear-x(s)$同理。

Reflection

使用$reflect-y$将$[-1 \ 0]$和$[0 \ 1]$ 作为x和y，则x方向反向，y方向不变。

组合变换

因为矩阵的乘法有结合性(associative)，所以可以变换步骤结合起来，但是因为矩阵乘法没有交换性(commutative)，所以变换结合的顺序将会影响最后结果。

  3. 旋转x轴和y轴回到v1，v2方向，即使用$R$矩阵变换。

这其实告诉我们对称矩阵的几何意义就是沿着任意轴的非等距缩放(nonuniform and non–axis-aligned transformation)。

  3. 旋转x轴和y轴到u1，u2方向，即使用$U$矩阵变换。

仿射变换（Affine Transformations）

通过上面的总结，发现无论如何都做不了平移的变换，因为如果在二维空间，(0, 0)与二维矩阵相乘，结果都是仍然是(0, 0)，这也是线性变换的特点，如果要进行平移变换，必须为矩阵多加一个维度，即齐次坐标(homogeneous coordinates)。

上如展示了一个四维空间，四维空间中点的坐标为(x, y, z, w)，如果限定w = 1，那么会落在图中那条w = 1的水平线上。将空间中的任意一点投影到w = 1上的做法是，将该点与原点连接起来，与w = 1的交点(x’, y’, z’, 1)为所求，根据三角形的相似性容易求得$x’ = x/w$。所以该点即为$(x/w, y/w, z/w, 1)$。而且，原点与该点连线所在的直线上的所有点在w = 1时都表示同一点。

建立齐次坐标，直接多添加一个分量，如在二维空间中，$(2, 3) -> (2, 3, 1)$，如果要重新得到笛卡尔坐标的表示方法，就将分量同时除以齐次分量，再舍弃齐次分量，如$(4, 6, 2) -> (2, 3, 1) -> (2, 3)$。也因为如此，可以发现，当齐次分量越大，笛卡尔表示的坐标值就越小，绘制的比例也就更小。(将上图的原点想像人眼或相机，w = 1平面想像成荧幕，投影到w = 1的过程即空间中物体投影到荧幕上)。

相机模型

有时候需要为某一个向量a创建一个正交基，比如对于相机，如果知道了它看向的方向，需要旋转这个方向来让相机看向别处，并不可以任意改变它的方向，这时根据相机看向的方向需要确定一个正交基(如下图所示forward, right, up建立的正交基)，让相机可以依据这个正交基的各个轴旋转，虽然根据一个向量不止一个结果。但是可以按照以下步骤构造出一个正交基。

1. 确定前向轴(forward axis) 。

其实这就是相机看向的方向。

2. 确定右边的轴(right vector) 。

现在只确定了一个向量，需要想办法求出与前向轴垂直的向量，解决办法是选一个任意向量(arbitrary vector)。这个向量和前向轴(forward axis)共面但不共线，求叉积后就可以得到与前向轴(forward axis)垂直的向量。

这个向量也不能任意选择，稍加思考发现选择(0, 1, 0)向量(可看作是朝向天空的方向)会比较好，这样无论前向轴朝向哪里，和这个任意向量组成的平面都和世界坐标的y轴垂直，根据这个向量计算出的右向量就朝向相机的右边。

注意：如果任意向量和前向轴共线了，那么右向量将无法求得。如果任意向量和前向轴很接近，那么求出的右向量就很不准确。

3. 计算向上向量(up vector) 。

既然已经确定了两个相互垂直的向量，因为根据叉积，得到了两个相互垂直的向量即可得到一个和这两个向量都垂直的向量，所以只需要求叉积即可。
